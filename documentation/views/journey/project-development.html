Fab Touché - Xduino shield

Inspired by Disney Research project - Touché, and the DZL arduino version (Low Resolution input data) of this "swept frequency capacitance sensor" is adapted to Fabable version to use plants as sensors.

Program

The logic of the Arduino IDE program of DZL is to change the clock on Timer 1 in order to generate sign waves between 1kHz and 3.5MHz. The main difference between Disney research technology and DZL arduino Touche is the resolution created by the hardware: Disney generates High resolution soft signal waves insted Arduino Touche square low resolution signals.

For my intension of the use of these technology in "Plant Stimuli installation" I don't need so much resolution.

For some recommendations is better to isolate and use one Xduino only for this sensor. In the logic of the micro controller the register clocks takes more of the micro controller resources.

Interface and Application programming

Processing - Visual Interface

The interface and the application programming was also based on DZL web pageand the library AdvancedTouchSensing from Illutron github account. The "Plant Stimuli App" receives the data in bytes transmited from the pre-programmed Xduino. Then with a sequency of commands the logic add different data to differents arrays. All these data arrays are used as a input to establish and configure via software, different gestures.

In the "Plant Stimuli App" I modified the code and re structure some of the functionallities: Implementing ControlP5 UI library for processing and Graphics are completele rebuild. All the original code files are maintained in this new version.

With the ControlP5 library I added some functionalities to interact and controll the light states.

Pure Data and AU Lab - Auditive Interface

I use Pure Data a AU Lab open source IDE's for sound composition. With PD I receive and map the values transmited from the Plant Stimuli App (processing) and then they transformed into a scales notes using algorithm composition "basic" techniques.

Processing and Pure Data communication

Since I am using the Serial Port to transmit the data from the Xduino Touche to Processing I use oscP5 library for processing following to applying some patches into Pure Data.. This library communicates via Serial using a IP Address. Any device can send and receive instructions and data using this communications. It was perfect for the project because I can separate and controll some of the large of amount of data for generate the graphics.
